Day 04 — Responsive Design

Mobile-first layouts that adapt to every screen

This exercise converts the Day 03 landing page into a fully responsive design using media queries, relative units, and fluid containers.

  • Mobile-first base styles

Fixed Widths

Responsive Feature Grid

Each card adapts from a single-column stack on mobile to a multi-column grid on wider screens, using auto-fit and minmax for fluid columns.

Media Queries

Breakpoints at 48rem and 64rem reshape layout, spacing, and typography for each device tier.

Relative Units

All spacing and font sizes use rem, %, and vw instead of fixed pixels for true scalability.

Fluid Typography

Headings scale smoothly with clamp() — no jumpy font size changes between breakpoints.

Mobile-First Nav

Navigation starts as a collapsible stack on small screens and expands to a horizontal bar on tablets.

Responsive Images

Images use max-width: 100% and height: auto to fit any container without overflow.

Container Queries

Fluid containers use min(), percentage widths, and auto margins to center content responsively.
